# Security Policy

## 🔐 Supported Versions

The following versions of Nitram are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| Latest  | ✅ Yes              |
| < Latest | ❌ No              |

We strongly recommend always using the latest version of Nitram.

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability in Nitram, please report it responsibly.

### 📩 How to Report

Please send a detailed report to:

- GitHub Issues: https://github.com/pantane1/Nitram/issues  
- Or contact directly via repository owner profile

### 📝 What to Include

To help us resolve the issue quickly, include:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Expected vs actual behavior
- Screenshots or proof-of-concept (if applicable)
- Browser and environment details

---

## 🔒 Responsible Disclosure

Please:

- Do NOT publicly disclose the vulnerability before it is resolved.
- Allow reasonable time for the issue to be investigated and fixed.
- Avoid exploiting the vulnerability beyond what is necessary to demonstrate it.

We are committed to addressing security issues promptly and transparently.

---

## 🛡 Security Practices

Nitram follows these security practices:

- Input validation and sanitization
- Protection against XSS (Cross-Site Scripting)
- Secure handling of user-generated content
- Regular dependency review
- HTTPS-only deployment (GitHub Pages)

---

## ⚠️ Scope

This security policy applies to:

- The official Nitram repository
- The live deployment: https://pantane1.github.io/Nitram/

Third-party services, browser extensions, or external integrations are outside the scope of this policy.

---

## 🙏 Acknowledgements

We appreciate responsible security researchers who help improve Nitram’s safety and reliability.

Thank you for helping keep Nitram secure.
